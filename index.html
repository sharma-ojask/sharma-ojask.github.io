WikiQA Span Extraction

Background
Extractive question answering models often struggle when transferred to new domains with limited labeled data. This project focused on adapting a SQuAD-trained model to the WikiQA dataset under an extremely low-resource setting.

My Contributions

  • Implemented tokenization, span alignment, and data preprocessing pipelines
  • Addressed noisy annotations and edge cases in the WikiQA dataset
  • Designed span-level evaluation metrics for extractive QA

Results
The domain-adapted model achieved an F1 score of 77.56%, outperforming the zero-shot baseline (72.56%) by a 5.0% absolute gain. Achieving this improvement with only 606 training examples demonstrates effective domain adaptation using high-quality annotations.

Bulk to the Future: Steelcase Capstone

Background
Steelcase’s transition to SAP S/4HANA required bulk packing across all U.S. warehouses. The IBM-provided redesign increased labor requirements and failed to scale efficiently. My classmates and I piloted redesign for Steelcase’s largest warehouse in Grand Rapids, serving as the template for nationwide rollout.

My Contributions

  • Designed the Virtual EOL using historical order completion patterns
  • Built a logistic regression model (87% weighted F1) to classify order duration
  • Created an operator-facing sorting cheat sheet for rapid decision-making

Results
Reduced labor needs from 12–13 operators to 4–8, cut bins per sublane from 28 to 17, freed 2,700+ sq. ft. of floor space, and delivered $300K–$600K in estimated savings.
